# 03 — Configuration & Project Structure Design

**Source:** `design.yaml` format, `src/metadata.js`, `src/filler.js`, `src/constants.js`

## Configuration File (`design.yaml`)

### Full Schema

```yaml
project:
  name: string # Project display name
  database: string # Database type (e.g., 'PostgreSQL')
  note: string # Project description
  extensionSchema: string # Schema for extensions (default: 'public')
  staging: [string] # Staging schemas (restrict import targets)
  dbdocs: # DBML document definitions
    { name }: # Each key → separate .dbml file
      include:
        schemas: [string]
        tables: [string]
      exclude:
        schemas: [string]
        tables: [string]

schemas: [string] # Schemas to CREATE

extensions: # Extensions to install
  - string # Simple: name only (uses extensionSchema)
  - name: string # Object: name + explicit schema
    schema: string

roles: # Database roles
  - name: string
    refers: [string] # Roles to GRANT to this role

import:
  options: # Default import options
    truncate: boolean # Truncate before import (default: true)
    nullValue: string # NULL representation (default: '')
  tables: # Tables to import
    - string # Simple: schema.table
    - name: string # Object: with overrides
      format: string # csv|tsv|json|jsonl
      truncate: boolean
      nullValue: string
  schemas: # Per-schema option overrides
    { schema }:
      truncate: boolean
      format: string
  after: [string] # Post-import SQL scripts to execute

export: # Tables/views to export
  - string # Simple: schema.table (default: csv)
  - name: string # Object: with format override
    format: string # csv|tsv|json|jsonl
```

### Entity Discovery (Implicit)

Tables, views, functions, and procedures are NOT required in `design.yaml`. They are **discovered from the file system**:

```
ddl/
  table/{schema}/{name}.ddl       → entity { type: 'table', name: 'schema.name' }
  view/{schema}/{name}.ddl        → entity { type: 'view', name: 'schema.name' }
  function/{schema}/{name}.ddl    → entity { type: 'function', name: 'schema.name' }
  procedure/{schema}/{name}.ddl   → entity { type: 'procedure', name: 'schema.name' }
  role/{name}.ddl                 → entity { type: 'role', name: 'name' }
  extension/{name}.ddl            → entity { type: 'extension', name: 'name' }
```

If entities are also listed in `design.yaml`, config values **override** discovered values (via `metadata.merge()`).

## Project Directory Layout

```
project/
├── design.yaml                   # Configuration (required)
├── ddl/                          # DDL scripts (discovered)
│   ├── table/{schema}/{name}.ddl
│   ├── view/{schema}/{name}.ddl
│   ├── procedure/{schema}/{name}.ddl
│   ├── function/{schema}/{name}.ddl
│   ├── role/{name}.ddl
│   └── extension/{name}.ddl
├── import/                       # Data files (discovered)
│   ├── {schema}/{name}.csv|tsv|json|jsonl
│   └── {script}.sql              # Post-import scripts
└── export/                       # Generated by `dbd export`
    └── {schema}/{name}.csv|...
```

### File Extensions

| Context     | Accepted                          |
| ----------- | --------------------------------- |
| DDL scripts | `.ddl`, `.sql`                    |
| Import data | `.csv`, `.tsv`, `.json`, `.jsonl` |
| Export data | `.csv`, `.tsv`, `.json`, `.jsonl` |

## Entity Type System

### Constants (`constants.js`)

```javascript
typesWithSchema = ['table', 'view', 'function', 'procedure', 'import']
typesWithoutSchema = ['role', 'schema', 'extension']
allowedTypes = [...typesWithSchema, ...typesWithoutSchema]

defaultImportOptions = { format: 'csv', nullValue: '', truncate: true }
defaultExportOptions = { format: 'csv' }
```

### Entity Lifecycle

```
File discovery (scan ddl/)
  → entityFromFile(path)          # Parse path → entity stub
  → parseEntityScript(entity)     # Read file, extract refs + search paths
  → matchReferences(entities)     # Resolve refs to known entities
  → merge(scanned, config)        # Config overrides discovered
  → organize(entities)            # Topological sort by dependencies
  → validateEntityFile(entity)    # Check naming, files, schemas
```

## Configuration Normalization (`filler.js`)

`fillMissingInfoForEntities(data)` ensures:

- Each entity type array exists: `roles`, `tables`, `views`, `functions`, `procedures`
- Each entity has `refers: []` default
- Each entity has `type` property matching its category

This runs before `metadata.clean()` to guarantee array access is safe.

## Staging Schema Restriction

The `project.staging` array defines which schemas accept data imports. Import operations targeting non-staging schemas are rejected during validation. This prevents accidental data loading into production schemas.

## Data Import Flow

```
design.yaml import config
  │
  ▼
Scan import/ folder
  → entityFromImportConfig() per file
  → merge with config tables list
  → apply schema-specific options (import.schemas.{name})
  → apply global options (import.options)
  │
  ▼
Execution order:
  1. Truncate target table (if configured)
  2. \copy data file into table
  3. Execute post-import scripts (import.after[])
```

## DBML Document Configuration

Each `project.dbdocs.{name}` entry produces a separate `.dbml` file:

```yaml
dbdocs:
  base: # → design-base.dbml
    exclude:
      schemas: [staging, migrate]
  core: # → design-core.dbml
    include:
      schemas: [config]
```

Filtering via `entitiesForDBML(entities, config)`:

1. Start with all table entities
2. If `include.schemas` set: keep only matching schemas
3. If `include.tables` set: keep only matching table names
4. If `exclude.schemas` set: remove matching schemas
5. If `exclude.tables` set: remove matching table names
